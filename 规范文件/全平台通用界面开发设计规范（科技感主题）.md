提示：本规范面向个人在全平台（Web、移动端、桌面端）实现统一科技感界面风格的场景，实际使用时应结合具体产品定位、品牌要求与目标平台设计指南（如 Material Design、Human Interface Guidelines）进行适配。

## 使用说明

### 一、适配核心原则

1. 品牌与主题适配：在应用本规范前，应先明确品牌主色、辅助色与整体气质，科技感主题可以作为基础框架，但应允许在配色与组件细节上做品牌化调整。
2. 多端一致性：在 Web / 移动 / 桌面多端实现时，应优先保证信息架构与核心组件行为一致，再根据平台特性微调交互与动效。
3. 可访问性优先：在追求科技感视觉效果的同时，必须确保对比度、字体大小与动效节奏符合无障碍与可读性要求。

### 二、落地与维护方式

1. 设计落地：UI 设计在出图时应引用本规范中的色板、字体层级与组件规范，避免每个页面单独定义一套样式。
2. 组件沉淀：前端开发应基于本规范构建基础组件库（按钮、输入框、提示框、系统框、导航等），通过统一组件实现样式与交互的一致性。
3. 文档同步：当产品视觉或交互策略发生变化（如调整主色、修改按钮状态样式）时，应同步更新本规范并在版本记录中说明。

### 三、自查与培训

1. 自查频率：建议在每次大版本设计重构或新模块上线前，对照本规范检查界面是否存在风格偏差。
2. 新人培训：新加入的设计师与前端工程师在参与项目前，应阅读本规范并通过一两个典型页面的实现演练。
3. 工具辅助：推荐使用设计系统/样式检查工具（如 Figma 组件库、Storybook）辅助规范落地与回归检查。

#### 规范自查清单（界面开发设计）

- [ ] 页面主色、辅助色与状态色符合本规范定义，且在全站范围内保持一致
- [ ] 字体家族、字号层级与行高使用统一规范，无随意设置
- [ ] 核心组件（按钮、输入框、提示框、系统框、导航等）均来自统一组件库，无“野生组件”
- [ ] 常见状态（加载、空状态、错误页等）均按规范实现，文案与样式统一
- [ ] 深色模式与浅色模式在配色与对比度上均满足可读性要求
- [ ] 无障碍要求（键盘可达、焦点可见、文本对比度等）已在关键页面中验证
- [ ] 全局底部信息与宣传文案符合统一展示规范，无违规/夸大用语
- [ ] 界面性能（首屏加载、动效流畅度）符合规范中的性能指引

# 一、文档总则

## 1.1 设计目标

打造全平台（Web、iOS、Android、桌面端）视觉统一、交互流畅的科技感界面，以明亮字体保障可读性，各功能模块采用独立界面设计，兼顾操作效率与视觉冲击力，传递精准、前沿的产品调性。

## 1.2 核心原则

- **一致性原则**：全局配色、字体层级、组件样式、交互逻辑统一，跨平台适配时保留核心视觉特征，仅适配系统原生交互差异。

- **科技感原则**：以线性元素、渐变光影、低饱和度冷色调为基础，搭配简洁动效，避免冗余装饰，凸显数字化、智能化特质。

- **可读性原则**：采用高对比度明亮字体，优化行高与间距，确保不同设备、光线环境下文本清晰可辨。

- **独立性原则**：各功能模块界面独立封装，布局、组件不相互依赖，支持单独调用与迭代。

## 1.3 适用范围

本规范适用于产品全端界面开发，涵盖提示框、输入框、系统框、按钮、表格、导航等所有核心组件及独立功能界面，是UI设计、前端开发、测试验收的统一依据。

# 二、基础设计规范

## 2.1 配色规范

### 2.1.1 主色体系

- **主色**：科技蓝（#1E88E5），用于核心按钮、导航栏、关键视觉元素，传递专业、前沿感。

- **辅助色**：青蓝（#00BCD4）、浅紫（#9C27B0），用于次要操作、状态标识、功能区分，增强界面层次感。

- **状态色**：成功（#4CAF50）、警告（#FFC107）、错误（#F44336）、禁用（#BDBDBD），统一用于全平台状态反馈。

### 2.1.2 中性色体系（保障字体明亮度）

- **背景色**：主背景（#FFFFFF）、次级背景（#F5F7FA）、深色背景（#263238）（用于夜间模式/强调区域）。

- **字体色**：主文本（#212121）、次文本（#757575）、辅助文本（#9E9E9E），确保文本与背景对比度≥4.5:1，大标题≥3:1。

## 2.2 字体规范（明亮化核心）

### 2.2.1 字体选择

- Web端：思源黑体（Noto Sans SC）、微软雅黑，优先加载系统默认字体，保证兼容性。

- iOS端：SF Pro + 思源黑体（中文），遵循系统字体渲染规则。

- Android端：Roboto + 思源黑体（中文），适配不同厂商定制系统。

- 桌面端：思源黑体、微软雅黑，禁止使用自定义字体（除非品牌强需求），避免加载卡顿。

### 2.2.2 字体层级

字体系统统一划分为「大标题」「标题」「正文」「辅助文本」「微小文本」五个层级，用于控制全局信息层级和视觉节奏。各层级的推荐字号、字重、行高及典型使用场景请参见附录B《字体层级速查》，在具体页面实现时禁止随意新增自定义层级或偏离该映射关系。

### 2.2.3 页面主标题组件规范

- **命名约定**：Web 端页面级主标题统一使用 `h1.page-title`（或等价样式类），仅在每个独立功能页面中出现 1 次，作为该页面视觉层级最高的标题。
- **字号与字重**：Web/桌面端默认字号为 **28px**，字重 **Bold（700）**；移动端默认 24dp，字重保持 Bold（700），行高建议 **1.3–1.4 倍**，确保紧凑但不压迫。
- **颜色与对比度（浅色模式）**：主标题文字颜色统一为深色主文本（推荐 `#1E293B` 或 `#212121`），与主背景色 `#FFFFFF` 的对比度需 **≥ 4.5:1**，在复杂背景上必须叠加纯色或半透明遮罩后再叠字，禁止直接压在高噪点图片上。
- **颜色与对比度（深色模式）**：深色模式下主标题文字颜色统一为 `#E0E0E0`，与主背景 `#263238` 对比度同样需 **≥ 4.5:1**，禁止使用半透明浅色字体降低可读性。
- **间距与布局**：标题上方外边距可为 0，**下方外边距 4–12px**，与副标题/说明文字拉开一到两个字号的垂直间距；左右保持与页面主内容同一对齐线（例如内容区最大宽度 1440px 居中时，标题与内容共享同一容器）。
- **联动关系**：页面主标题下方如有副标题（如 `p.page-subtitle`），字号应使用"辅助文本"或"正文"层级（14–16px），颜色使用次文本色（如 `#64748B`），禁止与主标题使用相同字号和颜色，避免层级混乱。

### 2.2.4 标题样式实现规范（Web端）

- **DOM 结构约定**：业务页面顶部标题区域统一采用以下结构，便于全局样式复用与自动化测试定位：
  - 列表/详情类页面：`div.page-header > div.header-left > h2.page-title + p.page-subtitle`
  - 仪表盘快捷区标题：`div.quick-actions > h2`
  - API 文档/设置等信息页：`div.page-header > .header-content > h2 + p`
- **CSS 变量与颜色统一**：所有 `h1/h2.page-title` 及页面级标题必须使用主题变量 `color: var(--text-primary, #1e293b);`，禁止直接写死纯色值（如仅写 `#1e293b`），以确保在浅色/深色模式下均保持足够对比度与可读性。
- **字号与层级映射**：
  - 页面内容区主标题（对应 Vue 页面内的 `h2.page-title`）：字号固定为 **28px**，字重 **700**，行高 **1.3**，与导航栏标题（布局组件中的 `h1.header-title`）保持一档层级差。
  - 卡片/区块标题（如 `Dashboard` 中的卡片 `h2`/`h3`）：字号 **16px**，字重 **600**，颜色同样使用 `var(--text-primary, #1e293b)` 或等价主题变量。
  - 副标题/说明文本：字号 **14px**，字重 **400**，颜色为次文本色（`#64748B` 或 `var(--text-secondary)`），与主标题在视觉上明显区分。
- **间距规范（与实现对齐）**：
  - 页面级 `.page-header` 下方统一 `margin-bottom: 24px`，内部 `.page-title` 与 `.page-subtitle` 之间垂直间距 **4px**（如 `margin: 0 0 4px;`），与当前 `项目管理/漏洞扫描/Webhook 管理/系统设置/API 文档` 实现保持一致。
  - 仪表盘 `quick-actions` 标题下方统一 `margin-bottom: 16px`，用于拉开与卡片网格的间距。
- **深色模式适配要求**：
  - 所有标题颜色必须通过 CSS 变量链路参与暗色主题切换（如 `--text-primary`、`--text-secondary`），禁止直接在组件内写入固定深浅色值以绕过主题系统。
  - 在设计和实现中，优先通过调整背景色/卡片色来保证标题对比度，而不是降低标题不透明度或使用半透明颜色。

## 2.3 布局规范

### 2.3.1 适配规则

- 基础尺寸：Web端以1920×1080为基准，移动端以360dp（Android）、430pt（iOS）为基准（高度自适应），采用弹性布局（Flexbox/Grid），禁止固定像素值。

- 安全区域：iOS顶部避开刘海（≥44pt）、底部Home Indicator（≥34pt）；Android顶部避开状态栏（≥24dp）、底部导航栏（≥48dp）；Web/桌面端左右边距≥16px/24px，保障内容不贴边。

- 间距规范：全局基础间距为8px，按8px倍数递增（8/16/24/32px），模块间距≥24px，组件内间距≥16px，保证界面呼吸感与整洁度。

## 2.4 动效规范

- 核心动效：采用线性缓动（Ease-In-Out，缓入缓出），避免生硬过渡，时长控制在0.3-0.8秒，增强科技感的同时不分散操作注意力。

- 反馈动效：按钮点击微缩放（Scale 0.95）+ 亮度降低10%；加载状态用环形进度条（科技蓝渐变：#1E88E5→#4FC3F7）；转场用淡入淡出或侧滑（Web/桌面端）、底部弹出（移动端）。

- 限制：避免过度动效叠加，独立功能界面切换动效统一，低配置设备可降级关闭非核心动效。

## 2.5 响应式设计规范

### 2.5.1 断点定义

| 断点名称 | 屏幕宽度范围 | 典型设备 | 布局策略 |
| --- | --- | --- | --- |
| 超小屏（xs） | <576px | 小型手机竖屏 | 单列布局，底部导航，简化交互 |
| 小屏（sm） | ≥576px | 大屏手机竖屏 | 单列/双列混合，优化触控区域 |
| 中屏（md） | ≥768px | 平板竖屏 | 双列布局，侧边导航可折叠 |
| 大屏（lg） | ≥992px | 平板横屏/小型笔记本 | 多列布局，侧边导航展开 |
| 超大屏（xl） | ≥1200px | 桌面显示器 | 宽屏布局，支持多窗口并行 |
| 巨屏（xxl） | ≥1600px | 大型桌面显示器 | 充分利用空间，信息密度最大化 |

### 2.5.2 布局适配策略

- **移动端优先**：默认采用移动端设计，逐步向大屏扩展。

- **弹性网格系统**：使用12列网格系统，列间距16px，外边距16px（移动端）、24px（桌面端）。

- **容器宽度限制**：主内容区最大宽度不超过1440px，超大屏幕居中显示，两侧留白。

- **相对单位优先**：优先使用百分比、vw/vh、rem等相对单位，辅以媒体查询固定值。

### 2.5.3 图片与媒体适配

- **响应式图片**：使用srcset属性或picture元素，根据屏幕密度加载不同分辨率图片。

- **图标适配**：提供16px、20px、24px、32px多种尺寸，遵循2倍、3倍图规范。

- **视频与地图**：使用16:9或4:3比例容器，宽度100%自适应，高度按比例计算。

# 三、核心组件设计规范（独立功能适配）

## 3.1 按钮组件

### 3.1.1 类型及样式

- **主按钮**：科技蓝（#1E88E5）填充，圆角8px，字体16px（移动端16dp）、Bold，点击区域≥48px×48px（移动端）、56px×48px（Web/桌面端），用于提交、确认识别核心操作。

- **次按钮**：白色背景+1.5px科技蓝边框，圆角8px，字体16px、Regular，用于取消、返回等辅助操作，与主按钮形成视觉区分。

- **文本按钮**：无背景无边框，科技蓝字体，hover时显示浅蓝背景（#E3F2FD），用于查看详情、编辑等轻量操作，节省界面空间。

- **图标按钮**：24×24px线性图标（科技蓝，描边1.5px，遵循3.5.4图标组件规范），无文字，点击区域≥48px×48px，用于关闭、刷新等空间有限场景。

### 3.1.2 状态规范

按钮包含正常、hover（Web/桌面端）、点击、禁用四态；禁用状态透明度降至50%，不可点击且无动效反馈，避免用户误判。

## 3.2 输入框组件

### 3.2.1 基础样式

白色背景，默认1px灰色边框（#E0E0E0），聚焦时切换为科技蓝（#1E88E5），圆角8px；高度56px（Web/桌面端）、52dp（移动端），左右内间距16px，字体为16px明亮主文本色（#212121）。

### 3.2.2 类型及交互

- **普通输入框**：占位文字为14px浅灰色（#9E9E9E），输入时实时校验；错误状态边框变红（#F44336），下方显示14px红色错误提示，精准告知问题。

- **密码输入框**：默认隐藏输入内容（●），右侧配置20×20px显示/隐藏图标（遵循3.5.4图标组件规范），点击切换状态；支持密码强度实时提示（科技蓝进度条），引导用户设置安全密码。

- **带前缀输入框**：前缀为20×20px科技蓝线性图标或文本（如手机号区号），图标遵循3.5.4图标组件规范，与输入内容间距8px，明确输入类型。

### 3.2.3 独立界面适配

- **圆角处理**：表格整体圆角8px，表头左上角、右上角同步圆角，规避直角生硬感，贴合科技风柔和线条设计。

#### 3.5.3.2 交互规则

- **行交互**：hover时行背景变为80%透明度浅蓝（#E3F2FD），无动效延迟；选中行背景为浅蓝（#E3F2FD），左侧加3px科技蓝实线标识，与单选/复选框状态联动。

- **排序反馈**：可排序表头右侧加20×20px科技蓝线性排序图标（遵循3.5.4规范），默认隐藏、hover时显示；升序/降序图标区分明确，排序状态下表头字体变为科技蓝（#1E88E5），强化视觉提示。

- **滚动适配**：内容超出容器宽度时显示横向滚动条（科技蓝样式，宽度4px，圆角2px，滚动区域背景#F5F7FA）；Web/桌面端滚动条常驻，移动端滚动时显示、静止时隐藏。

#### 3.5.3.3 适配规则

- Web/桌面端：支持表头固定（滚动表体时悬浮）、左侧1-2列固定（右侧内容滚动），固定列右侧加1px阴影（rgba(0,0,0,0.08)），避免滚动模糊。

- 移动端：优先简化列数，非核心列可隐藏或下拉展开；无固定表头，横向滚动时表头同步移动；单元格内容过长时最多显示2行（超出部分省略号），长按可查看完整内容。

- 深色模式：表头背景#37474F，表体交替背景#263238、#2E3C43，文本色#E0E0E0，边框及图标颜色#4FC3F7，与5.1深色模式配色规范完全对齐。

### 3.5.4 图标组件

#### 3.5.4.1 基础样式规范

- **风格统一**：全局采用线性图标设计，贴合科技感主题，描边宽度统一为1.5px（与图标按钮、输入框前缀图标保持一致），所有图标圆角统一为2px，拒绝直角或不规则圆角。设计优先使用布尔运算构建图形，严格执行像素对齐，避免模糊失真，确保视觉统一性。

- **配色规则**：默认状态下，功能图标采用科技蓝（#1E88E5）；状态类图标（成功/警告/错误）对应文档2.1.1节状态色；禁用状态图标为#BDBDBD（透明度50%）；深色模式下统一调整为#4FC3F7，适配整体深色体系。

- **视觉平衡**：借助8px栅格辅助线校准，确保不同图标视觉重量一致；核心表意部分居中呈现，占比不低于图标总面积60%，避免元素拥挤或松散。

#### 3.5.4.2 尺寸体系规范

- **核心尺寸**：按场景划分固定尺寸，遵循跨平台倍数规则——16×16px（移动端16dp，用于状态标注、文本旁辅助）、20×20px（移动端20dp，用于输入框前缀、列表项辅助，与输入框组件规范匹配）、24×24px（移动端24dp，用于图标按钮、导航栏，与按钮组件规范匹配）、32×32px（移动端32dp，用于功能模块入口、弹窗标题旁）。

- **倍图适配**：设计源文件提供@2x、@3x倍图（基础尺寸整数倍），确保高清显示；倍图尺寸需满足iOS（4的倍数）、Android（8的倍数）系统网格要求。

- **点击热区**：独立图标按钮热区符合系统规范，iOS端≥44pt×44pt，Android端≥48dp×48dp，图标居中放置，间距均匀分布。

#### 3.5.4.3 交互与反馈规则

- **基础交互**：可交互图标hover时（Web/桌面端），颜色加深为#1565C0（科技蓝深色调）；点击时伴随0.3秒线性缓动微缩放（Scale 0.95），与按钮组件反馈动效保持一致。

- **状态反馈**：加载状态图标旋转动效（1.5秒/圈，线性缓动）；切换类图标（展开/收起）动效时长0.3秒；选中状态可叠加科技蓝填充底色（圆角4px），增强标识性。

- **组合交互**：图标与文本组合时间距固定为8px（遵循全局间距规范），hover时同步变色，保持交互统一性。

#### 3.5.4.4 适配与资源管理

- **跨端适配**：Web/桌面端采用SVG格式（无损缩放），移动端优先矢量图；iOS端适配系统渲染规则，Android端适配厂商定制系统差异。

- **命名规范**：采用"组件_类别_功能_状态"格式（全小写英文字母，下划线分隔），示例：icon_button_refresh_default@2x.png、icon_input_password_visible.svg，便于开发调用。

- **资源归档**：按尺寸、状态分类归档，提供设计源文件（AI/Sketch）与成品图，源文件按"组件-平台-状态"分层管理，确保设计与开发资源一致。

#### 3.5.4.5 SVG 设计与实现规范（Web 端）

- **基础结构与坐标体系**
  - 所有 Web 图标资源必须使用 `SVG` 矢量格式，根节点统一包含 `viewBox` 属性，禁止仅通过 `width/height` 控制尺寸（LOGO、插画类品牌图形除外，可同时设置 viewBox 与固定尺寸）。
  - 通用功能图标需严格按照 3.5.4.2 中的核心尺寸在 24×24 或 20×20 网格上设计，确保图形在对应像素大小下无模糊失真；描边对齐像素网格，避免 0.5 像素位移。
  - 所有线性图标默认描边宽度为 **1.5px**，与按钮、输入框前缀图标保持一致；圆角、圆弧统一使用 2px 左右的圆角半径，贴合科技感线性风格。

- **配色与主题适配**
  - 功能类图标禁止在 `SVG` 内直接写死品牌色值（如 `fill="#1E88E5"`），应优先使用 `currentColor` 或 CSS 变量（例如 `var(--icon-primary, #1E88E5)`），由外层组件控制主题与状态色。
  - 状态类图标（成功/警告/错误）需与 2.1.1 节状态色保持完全一致，仅通过外层样式切换颜色，不在 SVG 内硬编码状态色；深色模式下同样通过主题变量切换。
  - 品牌级 LOGO（如 `CYP-Registry` 主标识）若需使用专属渐变或特定深色背景，可在附录或品牌规范中单独登记，说明其为**品牌例外**，不强制沿用 `currentColor` 方案。

- **命名规范与文件组织**
  - SVG 文件命名需与 3.5.4.4 中的资源命名规则保持一致，推荐格式：`icon_类别_功能_尺寸_状态.svg`，例如：`icon_system_lock_24_default.svg`、`icon_nav_dashboard_20_active.svg`。
  - 同一功能图标在不同尺寸（如 20/24/32）下应保证形态与语义一致，仅在细节上做微调；文件按组件/模块归档，避免散落各处。

- **导出与优化要求**
  - 从设计工具导出 SVG 时，应清理多余元数据（如编辑器专用 `metadata`、`sodipodi`、`inkscape` 命名空间等），保证文件结构简洁，利于版本管理与代码审查。
  - 禁止在 SVG 中嵌入栅格图片（`<image>` 标签）或冗余的隐藏图层；推荐通过自动化工具（如 SVGO）在构建阶段统一压缩、归一化属性顺序。
  - 动效相关属性（如复杂 `animate`/`animateTransform`）不建议直接写入图标资源文件，优先通过 CSS 或组件逻辑控制动效，保持基础图标资源的静态、可复用特性。

- **无障碍与语义规范**
  - 作为**纯装饰性图标**（仅增强视觉、不承载语义）的 SVG，在组件层需设置 `aria-hidden="true"`、`focusable="false"`，避免屏幕阅读器重复朗读。
  - 作为**承载语义的独立图标**（如错误页主图、空状态插画），应在 SVG 内提供 `<title>` 和必要的 `<desc>`，并通过 `role="img"` 与 `aria-labelledby` 关联，描述清晰简洁，与界面文案保持一致。
  - 同一功能图标在不同页面中语义标注需保持一致（例如统一使用"刷新"、"返回上一页"等描述），确保屏幕阅读器体验一致，符合第七章无障碍设计规范。

### 3.5.5 触摸反馈规范（移动端专项）

- **基础反馈**：移动端可点击元素（按钮、图标、列表项）触发触摸时，需提供系统原生震动反馈（轻触震动时长10ms），同时伴随视觉反馈（如按钮微缩放、列表项背景变浅蓝#E3F2FD，时长0.3秒），与Web/桌面端hover效果形成跨端呼应。

- **手势反馈**：滑动操作（如删除、筛选）时，边缘需显示科技蓝渐变引导线（渐变#1E88E5→透明）；下拉刷新触发时，加载图标为科技蓝环形进度条（与2.4动效规范一致）；上拉加载更多时，提示文本为14px辅助文本色（#9E9E9E）。

- **限制要求**：非可点击区域禁止添加触摸反馈，避免用户误判；连续触摸（如双击放大）反馈间隔≥300ms，防止触发冲突。

### 3.5.6 选择框（下拉式）组件

#### 3.5.6.1 基础样式

- **输入区域**：与输入框组件样式完全一致（高度56px/Web桌面端、52dp/移动端，白色背景、1px灰色边框#E0E0E0，聚焦时变科技蓝#1E88E5，圆角8px），左侧文本为16px主文本色#212121，右侧配20×20px线性科技蓝下拉图标（遵循3.5.4图标规范，描边1.5px），图标与文本间距8px。

- **下拉面板**：白色背景，1px灰色边框#E0E0E0，阴影0 4px 12px rgba(0,0,0,0.08)（与系统框阴影一致），圆角8px；面板宽度与输入区域一致，选项高度44px/Web桌面端、40dp/移动端，选中项背景为浅蓝#E3F2FD，左侧加3px科技蓝实线标识。

#### 3.5.6.2 交互规则

- 点击输入区域展开面板（伴随0.3秒淡入动效），点击空白区域或再次点击输入区域收起；支持键盘上下键切换选项，Enter键确认选中，Tab键跳出组件。

- 选项hover时（Web/桌面端）背景变浅蓝#E3F2FD，移动端触摸时触发10ms震动反馈+背景变色（与3.5.5触摸反馈规范一致）；多选模式（可选）支持勾选框（遵循3.5.2复选框规范），选中后文本右侧显示勾选图标。

- 支持模糊搜索（可选），搜索框嵌入面板顶部，样式与普通输入框一致，输入时实时过滤选项，无匹配结果时显示14px辅助文本"无匹配选项"。

### 3.5.8 卡片组件

#### 3.5.8.1 基础样式

- **核心样式**：白色背景，1px灰色边框（#E0E0E0），阴影0 2px 8px rgba(0,0,0,0.06)（比系统框阴影更轻薄），圆角12px；内间距遵循全局规范，上下24px、左右16px，模块间距≥16px（Web/桌面端）、12dp（移动端）。

- **结构规范**：支持"标题区+内容区+操作区"三层结构，标题区高度44px（Web/桌面端）、40dp（移动端），标题字体为"标题"层级（20px/18dp，Semibold），主文本色#212121，右侧可搭配24×24px图标（遵循3.5.4规范）；内容区字体为"正文"层级，操作区可放文本按钮/图标按钮（遵循3.1规范）。

- **适配变体**：可配置边框高亮样式（选中时1.5px科技蓝边框#1E88E5）、hover效果（Web/桌面端阴影放大为0 4px 12px rgba(0,0,0,0.08)，0.3秒缓动），适配列表展示、数据卡片等场景。

#### 3.5.8.2 交互与适配

- **可点击卡片**：hover时（Web/桌面端）背景加浅蓝底色#E3F2FD（透明度50%），点击反馈与按钮组件一致（0.3秒微缩放Scale 0.98）；移动端触摸时触发10ms震动反馈（遵循3.5.5规范）。

- **跨端适配**：Web/桌面端支持卡片hover展开详情（0.3秒淡入），移动端支持上下滑动卡片切换内容；卡片内容超出高度时，显示科技蓝样式滚动条（遵循3.5.3表格组件滚动条规范）。

### 3.5.9 标签组件

#### 3.5.9.1 基础样式

- **核心尺寸**：高度28px（Web/桌面端）、24dp（移动端），宽度自适应内容（最小宽度48px），圆角14px（半圆角），内间距左右12px、上下4px，字体为"辅助文本"层级（14px/14dp，Regular）。

- **类型变体**：

  - **实心标签**：科技蓝#1E88E5填充，白色字体，用于核心分类、状态标识；

  - **空心标签**：白色背景+1px科技蓝边框，科技蓝字体，用于次要分类、可移除标签；

  - **状态标签**：复用2.1.1状态色（成功/警告/错误），对应颜色填充+白色字体，用于结果反馈（如"已完成""待处理"）。

#### 3.5.9.2 交互规则

- **可移除标签**：右侧搭配16×16px线性关闭图标（同色系，描边1.5px），图标与文本间距4px；点击图标触发删除动效（0.3秒淡入淡出+轻微收缩），可搭配提示框二次确认（遵循3.3规范）。

- **联动适配**：标签组排列时，间距8px，超出容器宽度时支持横向滚动（无滚动条，Web/桌面端hover边缘显示渐变引导，移动端触摸滑动）；与卡片、表格联动时，标签位置居右或居下，不遮挡核心内容。

### 3.5.10 进度条/进度环组件

#### 3.5.10.1 基础样式

- **进度条**：高度8px（Web/桌面端）、6dp（移动端），背景色#E0E0E0，进度条为科技蓝渐变#1E88E5→#4FC3F7，圆角4px；支持分段样式（每段间距4px，对应不同进度节点），适配任务分步、文件上传进度展示。

- **进度环**：直径48px（Web/桌面端）、40dp（移动端），描边宽度6px/5dp，背景描边#E0E0E0，进度描边同科技蓝渐变，圆角描边（两端平滑）；中心可嵌入文本（"辅助文本"层级，科技蓝字体），显示百分比或进度数值，适配加载进度、完成率展示。

#### 3.5.10.2 交互与动效

- **进度动效**：进度变化时伴随0.5秒线性缓动，避免跳跃式更新；加载中状态（不确定进度）时，进度条/进度环触发"呼吸动画"（渐变颜色亮度循环变化，周期1.5秒），与2.4动效规范保持一致。

- **状态适配**：进度100%时，进度条/进度环末端显示24×24px成功图标（#4CAF50，遵循3.5.4规范）；失败时进度条变为错误色#F44336，末端显示错误图标，支持点击重试（联动3.1文本按钮规范）。

## 4.2 宣传词语规范

全平台界面中的宣传词（Slogan/Tagline）需根据项目定位、功能特性、用户场景进行差异化设计，确保宣传语精准传递产品价值，强化品牌认知。本规范定义宣传词的分类体系、编写原则及在各场景的具体应用。

### 4.2.1 宣传词分类体系

按项目类型和功能模块，将宣传词分为以下四个层级，每个层级对应不同的使用场景和表达策略：

| 层级 | 分类维度 | 核心特征 | 适用场景 |
| --- | --- | --- | --- |
| **L1-项目级** | 按产品线/项目定位 | 体现整体产品愿景，字数4-12字，简洁有力 | 启动页、登录页顶部、官网首页 |
| **L2-模块级** | 按功能模块划分 | 突出模块核心价值，字数6-16字，功能导向 | 功能入口页、模块切换页、侧边导航Tooltip |

### 4.2.2 编写原则

所有层级宣传词需遵循以下核心原则，确保内容质量与用户体验一致：

- **精准性原则**：宣传词必须准确反映项目/模块的核心功能或价值，禁止夸大或偏离实际能力，避免用户预期与产品体验产生落差。

- **简洁性原则**：控制字数在合理范围内（项目级4-12字、模块级6-16字、场景级4-12字），拒绝冗长表达，确保用户一眼能理解核心信息。

- **一致性原则**：同一项目或模块在不同界面使用统一的宣传词，避免信息混乱，强化用户心智。

- **科技感原则**：贴合本规范的科技感主题，优先使用现代、简洁的词汇（如"智能""精准""高效""协同"），避免口语化或过于文艺的表达。

- **无障碍原则**：避免使用生僻字、专业术语（如需使用需搭配说明），确保不同知识背景用户均可理解，宣传词与背景对比度≥4.5:1。

### 4.3 独立声明与数据处理界面规范

#### 4.3.1 使用场景

- **首次登录后展示**：用户首次使用产品并登录成功后，自动跳转至「个人声明与数据处理规范」独立界面，完整展示与当前产品适配后的《个人全平台通用声明与数据处理模板》核心内容。
- **只显示一次规则**：同一终端、同一账号在当前规范版本下仅需确认一次；前端通过本地存储键（例如 `cyp_registry_statement_ack_v2.1`）记录用户已阅读/同意状态，后续登录直接进入功能首页。
- **版本更新重新确认**：当声明模板版本升级（如从 v2.0 升级至 v2.1）时，应同步更新本地存储键名或版本号，触发用户重新阅读并确认。

#### 4.3.2 布局与样式

- **页面主标题**：使用 `h1.page-title` 组件，文案推荐为「个人声明与数据处理规范」，字号 28px、Bold（700），遵循 2.2.3 节页面主标题规范。
- **副标题说明**：在主标题下方使用 `p.page-subtitle` 说明阅读目的，如「请在继续使用 CYP-Registry 前，仔细阅读并确认以下声明内容。」字号 14–16px、次文本色，拉开 4–12px 垂直间距。
- **内容区域**：核心声明内容采用卡片式布局，单列滚动展示；重要条款（如免责范围、数据处理原则、用户权利等）使用小标题 + 分段文本，关键句可加粗强调，避免大段密集文字墙。
- **操作区**：页面底部操作区提供主按钮「我已阅读并同意，进入控制台」，按钮样式遵循 3.1 节主按钮规范；如需允许用户暂缓确认，可增加次按钮「稍后再说」，返回行为需与业务逻辑确认（如返回登录页或仪表盘）。
- **滚动与可读性**：长文本区域支持滚动阅读，行高 1.5–1.6 倍，两侧留白 ≥24px，移动端与桌面端保持一致层级与视觉节奏，确保阅读舒适。

#### 4.3.3 交互与状态

- **进入条件**：仅在用户认证成功且未确认当前版本声明时进入本界面；未登录用户访问该路径时，应跳转至登录页或提示先完成登录。
- **已确认状态**：若检测到当前版本已确认记录，则在进入该路由时自动重定向至首页/仪表盘，避免重复打扰用户。
- **反馈记录**：用户点击「我已阅读并同意」后，应立即写入本地存储并完成跳转；如后端需要合规记录，可同步上报确认时间与版本号，用于合规审计。
- **无障碍与键盘操作**：确认按钮需支持键盘 Space/Enter 触发，焦点态使用科技蓝描边；长文本区域支持键盘滚动（如 PageUp/PageDown），整体符合第七章无障碍设计规范要求。

# 五、特殊状态设计规范

## 5.1 加载状态规范

- **全局加载**：全屏遮罩（白色背景，透明度80%），居中显示科技蓝渐变环形进度条（直径48px，描边3px，渐变#1E88E5→#4FC3F7），进度条旋转动效1.5秒/圈（线性缓动）。

- **局部加载**：列表、表格、卡片等区域加载时，使用骨架屏占位，骨架屏背景为#F5F7FA，高亮区域为#E0E0E0（水平渐变动画，时长1.5秒，渐变#E0E0E0→#F5F7FA→#E0E0E0），避免空白等待；加载完成后平滑过渡到实际内容（0.3秒淡入）。

- **加载失败**：显示24×24px错误状态图标（#F44336，遵循3.5.4规范），下方文本16px主文本色（#212121）"加载失败"，下方配14px辅助文本"点击重试"（科技蓝，可点击，按钮样式遵循3.1按钮组件规范），点击后重新触发加载动效。

## 5.2 空状态规范

- **基础样式**：空状态图标为32×32px线性科技蓝图标（遵循3.5.4规范，如空列表、空搜索），居中显示；图标下方16px主文本色提示语（如"暂无数据""未搜索到结果"），下方可配14px辅助文本说明（如"请添加新内容"）；Web/桌面端与页面顶部间距≥64px，移动端≥48dp，确保视觉居中。

- **操作引导**：需用户触发操作的空状态（如空表单），下方添加主按钮（科技蓝，文本16px，遵循3.1按钮组件规范），按钮与文本间距24px，引导用户完成核心操作（如"添加数据"）。

## 5.3 表单验证规范

### 5.3.1 验证时机

- **实时校验**：用户输入过程中进行格式校验（如邮箱、手机号），错误即时反馈，但不阻断输入。

- **失焦校验**：用户离开输入框时触发校验，避免干扰输入流程。

- **提交校验**：表单提交时进行全量校验，所有错误一次性展示。

### 5.3.2 验证规则

| 验证类型 | 规则说明 | 反馈方式 |
| --- | --- | --- |
| 必填验证 | 字段为空时触发 | 边框变红，显示红色星号* |
| 格式验证 | 格式不符合要求（如邮箱、URL） | 边框变红，下方显示错误提示 |
| 长度验证 | 超出最大长度或不足最小长度 | 边框变红，显示剩余/所需字符数 |
| 唯一性验证 | 字段值已存在 | 边框变红，提示"该值已被占用" |
| 关联验证 | 依赖字段校验（如两次密码一致） | 边框变红，提示具体差异 |

### 5.3.3 错误提示规范

- **位置**：错误提示显示在输入框正下方，与输入框间距8px。

- **样式**：14px红色（#F44336）字体，错误图标（16×16px，遵循3.5.4规范）与文本间距4px。

- **动画**：错误提示出现时伴随0.3秒淡入动效。

- **清除**：用户开始修正错误时，错误提示延迟500ms消失。

### 5.3.4 成功反馈

- 表单提交成功后，显示成功提示框（浅蓝背景#E3F2FD，科技蓝图标，遵循3.3规范）。

- 自动跳转到目标页面或停留当前页面并清空表单（根据业务需求）。

## 5.4 错误页面规范

### 5.4.1 错误类型及设计

| 错误类型 | HTTP状态码 | 页面设计 |
| --- | --- | --- |
| 页面不存在 | 404 | 科技蓝404图标（32×32px），主标题"页面未找到"，副标题"您访问的页面不存在或已被移除"，提供返回首页按钮（主按钮）和刷新按钮（次按钮） |
| 服务器错误 | 500 | 科技蓝错误图标（32×32px），主标题"服务器错误"，副标题"请稍后重试或联系管理员"，提供刷新页面和返回首页按钮 |
| 无网络连接 | - | 科技蓝断网图标（32×32px），主标题"网络连接失败"，副标题"请检查您的网络设置"，提供检查网络按钮（图标按钮）和重试按钮（次按钮） |

### 5.4.2 统一规范

- **布局**：错误信息居中显示，与顶部间距≥64px（Web/桌面端）、48dp（移动端）。

- **按钮**：所有错误页面至少提供一个主按钮（返回/刷新），根据错误类型决定是否需要次按钮。

- **动效**：错误图标伴随0.5秒淡入动效，按钮hover效果遵循3.1规范。

- **深色模式**：错误图标调整为#4FC3F7，背景适配深色背景色。

# 六、深色模式专项规范

- **文本色**：主文本#E0E0E0、次文本#B0BEC5、辅助文本#78909C；

- **组件色**：按钮/图标主色#4FC3F7、边框色#455A64、选中背景#1976D2（科技蓝深色版）。

## 6.2 切换与适配规则

- **切换触发**：支持系统自动切换（跟随系统深色模式设置）与手动切换（开关组件控制，与3.5.1节开关规范一致），切换动效为0.5秒淡入淡出，避免视觉突兀。

- **组件适配**：所有组件（按钮、输入框、表格等）需同步适配深色配色，无漏改场景；图片资源适配深色模式时，降低亮度30%，确保不刺眼。

---

# 七、无障碍设计规范

## 7.1 核心要求

- **色彩无障碍**：所有文本与背景对比度严格遵循≥4.5:1（大标题≥3:1），禁止仅用颜色区分状态（如错误提示需同时搭配图标）；色盲友好配色，科技蓝主色在红绿色盲视角下可清晰识别。

- **屏幕阅读器适配**：所有组件（按钮、输入框、图标）需添加语义化标签（如"确认按钮""搜索输入框"），图标需补充文字说明；弹窗、提示框弹出时，自动触发屏幕阅读器播报内容。

- **操作无障碍**：可点击元素点击热区不小于48px×48px（移动端）、56px×48px（Web/桌面端）；支持键盘导航（Tab键切换焦点，Enter/Space键触发操作），焦点状态显示科技蓝边框（2px），清晰标识当前选中项；表格组件支持键盘上下键切换行选中状态，左右键切换单元格焦点。

---

# 八、全球化与多语言适配

## 8.1 文本与布局适配

- **多语言支持**：支持中文（简体/繁体）、英文，后续可扩展其他语言；文本外部化，所有界面文本不硬编码。

- **文本扩展**：预留足够文本扩展空间，英文文本长度约为中文的1.5-2倍，界面布局需适配文本增长。

- **RTL布局**：预留RTL（从右到左）布局支持，阿拉伯语、希伯来语等语言需镜像布局。

## 8.2 日期、时间、数字格式

| 语言 | 日期格式 | 时间格式 | 数字格式 |
| --- | --- | --- | --- |
| 中文 | YYYY年MM月DD日 | HH:mm:ss | 1,234.56 |
| 英文 | MM/DD/YYYY | HH:mm:ss AM/PM | 1,234.56 |
| 繁体中文 | YYYY年MM月DD日 | HH:mm:ss | 1,234.56 |

## 8.3 图标与符号

- **通用符号**：使用国际通用图标，避免文化特定符号。

- **颜色含义**：注意不同文化对颜色的理解差异（如白色在西方象征纯洁，在中国传统用于丧事）。

# 九、性能优化指引

## 9.1 渲染性能

- **减少重排重绘**：避免频繁读取布局属性，使用transform和opacity实现动画。

- **CSS选择器优化**：避免使用过于复杂的选择器，优先使用类选择器。

- **GPU加速**：为动画元素添加will-change: transform，提升渲染性能。

## 9.2 资源加载优化

- **图片优化**：使用WebP格式，懒加载非首屏图片，指定图片尺寸避免布局偏移。

- **字体优化**：使用font-display: swap，子集化字体文件减少加载体积。

- **代码分割**：按路由和组件拆分代码，路由懒加载，首屏资源最小化。

## 9.3 交互响应优化

- **点击响应**：确保点击事件在100ms内响应，长任务使用Web Worker或requestIdleCallback。

- **输入响应**：输入框实时搜索功能防抖（300ms），避免频繁请求。

- **动画帧率**：动画保持60fps，避免卡顿，低配置设备可降级关闭非核心动效。
